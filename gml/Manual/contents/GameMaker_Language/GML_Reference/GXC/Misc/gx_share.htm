gx_share

This asynchronous function is used on GX.games to share an image (as a buffer) or a URL (as a string) through the browser's share function.

The function returns 0 on success or -1 on error. When the request is complete, the Async - Social event will be triggered with the following keys in its async_load DS Map:

  • "type": "share_complete" (String)
  • "success": true or false
  • "error": Only if "success" is false, contains the error message returned by the browser

See the example at the bottom of the page for sharing images.

 

Syntax:

gx_share(content, [title], [text], [filename]);

Argument Type Description
content Buffer or String A buffer containing the image data (see example below) or a string containing the URL to share
title String  The title for the share dialog
text String  The text for the share diolog
filename String  The filename in case of an image (should contain the extension)

 

Returns:

N/A

 

Example:

var _buff_screenshot = buffer_create(1024, buffer_grow, 1);

surface_save(application_surface, _buff_screenshot);

var _filename = $"screenshot_{string_digits(date_current_datetime())}.png";
gx_share(_buff_screenshot, "Share screenshot", "Screenshot taken of Game", _filename);

buffer_delete(_buff_screenshot);

The above code creates a grow buffer and saves the application_surface into the buffer. As the application surface contains the final render of the game (excluding GUI), this essentially captures a screenshot of the game in the buffer.

After setting up a local variable with the file name (containing digits from the current Datetime), it calls the gx_share function to share that buffer as an image with the optional share dialog title, text and filename passed.

At the end it deletes the buffer as it's no longer needed in memory.
