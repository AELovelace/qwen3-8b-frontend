# Reporting Errors

While great care has been taken when creating GameMaker to ensure that it is a stable and reliable tool, there is always the possibility that an issue could occur given the nature of the software and the multiple operating systems it runs on.

Most IDE errors will be caught by GameMaker and you may see an error that GameMaker has gone "unstable", which also prompts you to save your projects and then close and restart the program.

If this happens, or you suspect that either the [Runtime](#) or the IDE contains a bug, then we would request that you file a bug report using the **Report A GameMaker Bug** option from [The Help Menu](../IDE_Navigation/Menus/The_Help_Menu.md). Please always give as much information as possible and ensure you fill in every field as best you can.

Please also strongly consider allowing the bug reporter to send us a copy of your project, as this may be essential to reproducing your problem \- we may be unable to help if you do not provide the project. All of your files are always attached privately when you send them using the IDE bug reporter.

### Report A Bug Window

The [Help](../IDE_Navigation/Menus/The_Help_Menu.md) \-\> **Report a GameMaker Bug** option will open a window where you can create a bug report from within the IDE. If you're not signed in, a message is shown to ask you to **Sign In/Register**. It's also possible to **Continue Anonymously**.  

If you are using a GameMaker account, it is recommended to connect your GitHub account to your GM account via the [GameMaker Account Dashboard](https://gamemaker.io/en/account). This will ensure your submitted reports are tied to your account, so you can add additional comments if you need to and also receive notifications from our team when the issue has been updated.
 

In the **Report a Bug** window you must fill the **Title** and **Description**. You should also enter **Steps To Reproduce** the issue; however, that is optional as you may not be confident in knowing how to recreate the problem. You must assign an **Issue Category** and if you select a category related to building projects or in\-game issues here, you will additionally be asked for the **Platforms Reproduced On** to say which platforms you know have this problem (and which do not).

You also have the option to **Privately attach this project**, which is generally required for many kinds of bug reports. You can also attach any additional files that may help your bug report.

Lastly, depending on your account subscription type, you may have the ability to set what the **Issue Visibility** of the bug report itself should be and therefore which bug database it is reported to.

  All files attached via the Bug Reporter are uploaded privately and are only accessible by the GameMaker team (even though the bug report itself is public).

  If you forgot to attach your project or any additional files with this dialog, you can use the **Upload a Bug/Ticket Sample** option in the [Help menu](../IDE_Navigation/Menus/The_Help_Menu.md) to upload those files and get a link that you can add to your issue.

  If uploading a bug report fails, GameMaker may retry uploading without your project included, if you have opted to include your project. In this case you should use the option mentioned in the **TIP** above and include it in a comment on the bug report.

As mentioned in the window, you should first check the list of [Known Issues](https://github.com/YoYoGames/GameMaker-Bugs/issues) before submitting. You also have the possibility to report a [security vulnerability](https://github.com/YoYoGames/GameMaker-Bugs/security/policy).

Finally, you can submit the report by clicking the **Submit** button.

  You may be shown a prompt to sign in again if you are logged in but the bug is unable to be submitted through your GitHub account and is being sent anonymously. You have the option to continue sending the bug anonymously or sign in again to attempt posting through your GitHub account.

### Crash Detected Window

In extreme cases where GameMaker crashes entirely, you may see on your next startup that the IDE bug reporter tells you that an earlier crash was detected and it will ask you to send us a bug report. Please do this.

Clicking on "**Report"** will open the "**Report a Bug"** window described above. You can fill in any information you want to include with the report and click on "Submit".

If you had a project open when the IDE crashed, you will be asked, after clicking "Submit", if you want to include that project privately with the bug report:

To know exactly what data is being sent, then please see [this article](https://github.com/YoYoGames/GameMaker-Bugs/wiki/Reporting-Bugs-Via-The-IDE#what-actually-gets-sent-in-a-bug-report) on the GameMaker wiki.
